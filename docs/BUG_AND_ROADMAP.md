# 缺陷与需求梳理 / 解决顺序排期

> 来源：产品与用户反馈整理（持续更新）  
> 说明：**BUG** 为缺陷或错误行为；**需求** 为功能增强或非缺陷类诉求。排期优先：**安全 → 阻断使用类缺陷 → 性能 → 大型能力**。

---

## 一、BUG（缺陷 / 错误行为）


| 编号     | 简述                                       | 说明                            | 状态 |
| ------ | ---------------------------------------- | ----------------------------- | -- |
| **1**  | 角色授权：动子权限后丢父菜单，入口消失；分配后左侧能显示，点进页面后左侧导航没了 | 保存时补全祖先菜单 id；授权树节点用 `value ?? id` 建父链（`PermissonDrawer`） | **已验证解决** |
| **2**  | 左侧 / 顶部 / 混合三种布局下：兄弟目录同时高亮、或父级不亮、混合布局展开被收起等 | **`MenuItem.menuItemIndex`**：目录（有可见子节点）的 `el-menu` **index** 仅用菜单树 `path.resolve` 结果，禁止对目录用 `router.resolve(name)`（易解析成父级短路径如 `/task`，触发兄弟分支误亮）；**叶子** `index` 用 `router.resolve(name).path`，与 `route.path`、`default-active` 同源。**`BasicMenu`**：`default-active` = `meta.activeMenu` 或规范化 `route.path`；去掉 `el-menu` 按 `route.path` 的 **`:key`**（避免每次跳转整表重建）；`watch` 仅在 **`nextTick` 内一次** `updateActiveIndex`。**`MixLayout`**：`BasicMenu` 使用 **`:key="mixSideMenuKey"`**，仅切换顶级模块时重建侧栏。父级标题高亮依赖 **Element Plus** `el-sub-menu.is-active`。另：目录 **redirect** 必填见前后端校验（数据规范）。 | **已验证解决** |
| **7**  | 混合布局下点「AI 智能助手」会跳到应用管理                   | 顶栏当前模块与路由前缀误匹配（如 `/app` 与 `/application`）；已用最长前缀匹配修复 | **已验证解决** |
| **8**  | 历史对话内容不全（仅首条）、带附件对话异常                    | AI 对话/附件链路 BUG                | — |
| **14** | 建普通用户分配权限：未全选二级时一级消失，二级变一级               | 与 **#1** 同类；随 **#1** 一并修复与验证        | **已验证解决** |
| **15** | 先查库再分页，应改为数据库分页                          | 性能 + 大数据量正确性风险                | — |
| **21** | 代码生成不识别表主键，强制 `id`                       | 代码生成 BUG                      | — |
| **24** | 登录慢，因查 IP 等，希望优化                         | 性能类缺陷（体验上等同 BUG）              | — |
| **27** | 新增用户密码明文传输                               | **安全漏洞**，需 HTTPS + 传输层防护或密码策略 | — |
| **28** | PG 下 date 类型新增报错                         | 反馈已解决，保留作回归验证                 | — |

## 四、建议解决顺序（排期）

### 第一阶段（优先：安全 + 阻断使用 + 明确 BUG）

1. **#27 密码明文传输** — 安全最高优先级（HTTPS、前端摘要/后端策略、接口审计）。
2. ~~**#1 + #14 权限与菜单树**~~ — **已验证解决**（回显、保存补祖先、`menuTreeNodeId`）。
3. ~~**#7 混合布局 + AI 助手跳转错误**~~ — **已验证解决**（`MixTopMenu` 最长前缀匹配顶栏路径）。
4. **#21 代码生成主键** — 影响生成代码正确性，改动面相对可控。
5. **#28 PG date** — 已解决则做回归用例即可。

### 第二阶段（体验与性能）

1. **#24 登录慢（IP 等）** — 异步、缓存、可配置关闭等。
2. **#15 数据库分页** — 按模块/接口分批改，优先高频列表。
3. **#6 菜单多卡顿** — 前端虚拟列表/懒加载 + 接口按需加载 + 后端必要索引。
4. ~~**#2 三种布局下菜单高亮与混合侧栏展开**~~ — **已验证解决**（见 BUG 表 **#2** 说明）。
5. **#8 AI 历史与附件** — 独立排 AI 子迭代。

### 第三阶段（规范与国际化）

1. **#26a 统一 order 排序** — 约定 + 扫描替换高风险点。
2. **#10 中英文** — 可持续迭代，可先列文案清单再分批补全。

### 第四阶段（架构与集成类，需立项/分期）

1. **#11 存储抽象（OSS/S3/FTP…）**
2. **#12 Python 3.13 + 依赖升级** — 全量测试后再上生产。
3. **#22 WebSocket 通知机制**
4. **#25 免登录开关**
5. **#3、#4、#17** 企微/飞书/钉钉/OA 同步（常分多期：登录 → 组织 → 消息）

### 第五阶段（大功能，单独路线图）

1. **#13 多租户**
2. **#9 菜单级数据权限**（与 #5、#4 联动）
3. **#16 AI 知识库/Skills**
4. **#18 工作流 + 应用编排**
5. **#19 BPM 表单**
6. **#20 菜单导入导出升级**
7. **#23 博客 Demo**
